# WC26 Predict — Commercial Readiness

This document defines how WC26 Predict should be presented and prepared for future commercialization.

---

## 1. Commercial thesis

WC26 Predict should be commercialized as a football research and content workflow product, not as a betting product.

Best-fit directions:

- creator research workspace
- match analysis report generator
- football data dashboard
- AI-assisted content preparation tool
- post-match review and model learning system
- educational AI sports analytics project

Avoided directions:

- betting tips
- bookmaker affiliate funnel
- gambling prediction signals
- odds arbitrage
- paid picks
- guaranteed score prediction

---

## 2. Potential product packages

### 2.1 Creator Studio

For short-video creators, newsletter writers, football bloggers, and community operators.

Core value:

- one-click match background
- team ranking material
- form and context summary
- safe script outline
- visual chart material
- post-match review notes

### 2.2 Analyst Workspace

For users who care about models, assumptions, and evaluation.

Core value:

- model snapshots
- probability calibration research
- Brier / RPS review
- model version tracking
- data provenance
- source-linked signals

### 2.3 API / Internal Tooling

For teams that want structured football analysis in their own workflow.

Core value:

- match analysis endpoint
- report generation
- safe-output mode
- source audit
- scheduled snapshots

---

Suggested subheadline:

> WC26 Predict turns football data, model evaluation, match context, and source-based signals into structured research reports for analysts and creators.

Suggested CTA:

> Explore the GitHub repo

or:

> View the demo workflow

---

## 4. What to show publicly

Good public demos:

- dashboard screenshots
- safe match report examples
- model pipeline diagram
- data provenance panel
- post-match review example
- "how it was built with AI" story

Do not publicly show:

- bookmaker names
- raw odds
- betting-market tables
- betting calls to action
- "hit rate" marketing
- "guaranteed" prediction language

- Add audit logs

---

## 6. Commercial risk controls

Before any monetization:

- public reports must pass forbidden-term scan
- output mode must default to `public_safe` or `creator_safe`
- internal market-consensus data must not leak to public UI
- no gambling-oriented SEO pages
- no bookmaker affiliate links
- no "buy picks" language
- no exaggerated accuracy claims

---
