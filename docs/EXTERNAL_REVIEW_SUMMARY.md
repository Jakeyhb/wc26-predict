## 一、项目概要

**WC26 Predict** 是一个 2026 年世界杯比赛预测系统。使用 7 组件序贯融合流水线，对每场比赛输出 H/D/A（主胜/平局/客胜）概率 + 比分分布。

**当前阶段**: 小组赛 48 场已结束，淘汰赛 R32（Round of 32）已打 7 场，剩余 9 场（7月1-3日）。

**核心技术栈**: Python 3.11 + SQLite + scikit-learn + 自研统计模型

**运行模式**: 手工触发预测和复盘（非自动），通过 Claude 代理执行 7 步标准流程。

## 二、系统架构

### 2.1 7 组件序贯融合流水线

```
DC(90%) → Enhancer(10%) → Weibull(10%) → NegBin(5%) → Elo(22%) → Pi(18%) → Market(20%)
```

括号内为**名义权重**。因为是序贯融合（非并行加权平均），每个组件的**有效权重**不同——靠前的组件（DC）实际影响力远大于名义权重，因为下游组件只能在剩余概率空间内调整。

| 组件 | 类型 | 作用 | 当前表现(20场) |
|:---|:---|:---|:---|
| **DC** (Dixon-Coles) | 统计模型 | 双变量泊松，核心 xG 产出 | 方向正确 70% |
| **Enhancer** | 机器学习 | Tabular ML 增强特征 | 方向正确 30% |
| **Weibull** | copula 模型 | 极端比分检测，over/under | 30% 失败率，经常被跳过 |
| **NegBin** | 统计模型 | 过度离散修正 | 方向正确 ~50% |
| **Elo** (κ-Davidson) | 评级系统 | 实力评分 + 平局建模 | 方向正确 60% |
| **Pi-Rating** | 评级系统 | 基于赛果的评分 | 方向正确 65% |
| **Market** | 市场数据 | 多博彩商共识（≥8家） | 方向正确 **80%** 🥇 |

融合后还经过：**Isotonic 校准器**（69 场 WC26 训练）→ **KO 平局结构性修正**（淘汰赛平局 ×1.15-1.18 乘数 + 18-22% 地板）。

### 2.2 关键参数

| 参数 | 值 | 说明 |
|:---|:---|:---|
| DC 名义权重 | 0.90 | Enhancer 太差 (30%)，用高 DC 权重压制 |
| Market 最大权重 | 0.30 | 博彩商方向正确率最高，但设上限防过拟合 |
| Elo/Pi 权重 | 0.22 / 0.18 | 淘汰赛专用 |
| Weibull 跳过阈值 | 偏离均值 >2σ | 极端输出 (70%+) 自动跳过 |
| KO 平局地板 | 18-22% | 淘汰赛平局率高于小组赛 |
| 最低概率裁剪 | 2% | 防止 0% 极端输出 |

### 2.3 数据库结构

**SQLite** (`backend/data/local_stage2.db`)，约 15 个业务表：

- `wc26_schedule` — 赛程（104 行，61 场 FINISHED）
- `match_results` — 比赛结果（比分、进球时间）
- `prediction_snapshots` — 预测快照（组件概率、融合结果）
- `prediction_runs` — 预测运行记录
- `postmatch_eval` — 赛后评估（Brier/LogLoss/RPS）
- `prediction_learning_log` — 自进化学习日志
- `elo_ratings` / `pi_ratings` — 评级历史

## 四、赛后复盘流程（当前 7 步）

每场比赛结束后，人工触发：

1. **结果确认** — Web 搜索 ≥2 个独立源验证比分
2. **DB 更新** — 写入 match_results 表
3. **组件评估** — 逐组件计算 Brier/LogLoss，方向正确性
4. **LOO 边际贡献** — 留一法计算每个组件对融合 Brier 的边际贡献
5. **自进化决策** — 是否调整融合权重或其他参数
6. **报告生成** — Markdown 报告存到 `reports/postmatch/`
7. **Memory 更新** — 更新项目状态文件 + 记忆系统

### 当前复盘流程的核心缺陷

**复盘只分析"预测概率 vs 比分"，不分析比赛过程。**

以 MEX-ECU 为例，赛后复盘共 247 行，但我们**至今不知道**：

- 墨西哥实际射门多少次？射正几次？
- 实际控球率是多少？
- 厄瓜多尔的防守行为（抢断、解围）是否支持"防守强"的判断？
- 墨西哥的进攻是如何打穿厄瓜多尔的？（定位球？反击？阵地战？）

我们事后判断"DC xG=0.53 是严重低估"，但这个判断**完全基于比分（2-0）+ 小组赛记忆（MEX 场均 2 球）**，没有任何这场比赛的实际过程数据支撑。

### 5.2 为什么这些数据对复盘至关重要

**用 MEX-ECU 失败案例说明：**

复盘报告中我们写了：

> "DC 混淆了墨西哥的防守强与进攻弱 — xG 预测 0.53，但墨西哥小组赛场均 2.0 球"

但如果我们有**这场比赛的实际赛况数据**：

```
假设我们能看到:
  射门:        MEX 14(6) — ECU 5(1)
  实际 xG:     MEX 1.80 — ECU 0.30
  控球率:      MEX 58% — ECU 42%
  传球成功率:   MEX 87% — ECU 74%
  角球:        MEX 8 — ECU 2
  禁区内触球:   MEX 22 — ECU 7
```

**一眼就能确认**：DC 的 xG=0.53 完全错误，这是一场墨西哥碾压式胜利，与"两队接近→平局"完全相反。

而如果数据是：

```
假设另一种情况:
  射门:        MEX 8(2) — ECU 7(3)
  实际 xG:     MEX 0.60 — ECU 0.55
  控球率:      MEX 49% — ECU 51%
```

那说明墨西哥赢球有运气成分，DC 的 0.53 xG 和 50.5% 平局概率其实是合理的——错误的只是最终结果（比分），而不是建模逻辑。

**没有过程数据，我们无法区分"模型错了"和"模型对了但结果反常"。**

### 5.3 具体需要补的数据维度

**Layer 1 — 进攻产出（验证 xG 模型）**
- 射门（总计 / 射正）
- 重大机会（big chances）
- 禁区内触球
- 实际 xG（Opta/StatsBomb 数据源）

**Layer 2 — 比赛控制（验证实力差距判断）**
- 控球率
- 传球成功率
- 进攻三区传球
- 角球

**Layer 3 — 防守行为（诊断"防守强 vs 进攻弱"混淆）**
- 解围
- 拦截
- 抢断
- 扑救
- 犯规/黄牌

**Layer 4 — 体能/节奏**
- 跑动距离
- 冲刺次数
- PPDA（每次防守动作允许的传球数，反映压迫强度）

## 七、6 月 30 日-7 月 1 日三场复盘总结

### 关键发现

1. **Market 双面性定理**：CIV-NOR（Market 边际 -0.0788，因为数据质量差 Elo=1500 默认值）vs FRA-SWE（Market 边际 +0.0582，数据质量好），证明 Market 组件的贡献质量与数据质量直接相关。

2. **"Weibull 跳过"决策需要分场景**：FRA-SWE 的 84.4% 法国被错误跳过（事后 Brier=0.0421 全场最佳），说明在实力悬殊场景下，Weibull 极端值是信号而非噪声。但 MEX-ECU 的 70% 平局又确实应该被跳过。

3. **MEX-ECU 是淘汰赛最差预测**（Brier=0.6104），根源在于 DC 将墨西哥的"3 场零封"解读为"进攻乏力"而非"防守出色"。这是**模型输入解读错误**，不是概率融合问题。

### 自进化纪律

三场复盘中**全部选择了"不调整融合权重"**，坚持等到 16 场 R32 全部完赛后统一决策。这避免了基于小样本（7 场）做出错误调整。

### 问题 2：如何用赛况数据改进复盘

**背景**：当前复盘 = 预测概率 vs 最终比分。有了过程数据后，应该怎么做？

**具体问题**：
- 如何用实际 xG 来**诊断** DC 模型是否合理（即使比分结果碰巧对了/错了）？

- 有没有一套标准化的**赛况 vs 预测对比框架**？比如"xG 误差"、"控球率 vs 预期控球率"、"射门转化率异常"？
